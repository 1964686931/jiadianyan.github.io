问题情况

[单人]先写项目，后建仓库，push被拒绝

问题
如果建仓库时生成了readme文件，则涉及push被拒绝。因为本地仓库落后于远程仓库。

指令集（不合并仓库而强行push版本，有同名文件则会覆盖，比如根目录下的readme.md)

在项目文件夹中打开git bash
git init 生成.git文件夹
git add . 添加文件
git commit -m “” 提交
git remote add origin [git仓库地址] 关联远程仓库
git push -f 强制push如果是单人没什么问题，此时多人都强行push的话可能出现覆盖
如果提示…按照提示来，比如按照提示加一个参数：git push -f –save-upstream
